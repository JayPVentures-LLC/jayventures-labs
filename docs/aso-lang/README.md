# ASO-Lang

Status: v0.1 formal-language draft  
Lane: jayventures-labs research, validation, and experimentation
Owner: JayPVentures LLC  
Classification: formal language layer for ASO/IAT research

## Purpose

ASO-Lang defines a controlled vocabulary, symbol set, grammar, relationship verbs, namespaces, and inference rules for Adaptive Systems Ontology and companion frameworks.

The goal is to reduce ambiguity before mathematical formalization, measurement, simulation, or domain translation.

## Core rule

No equation, theorem, measurement, or simulation should use a term that lacks a stable language definition.

## Canonical files

- `ASO-LANG-v0.1.md` — formal language specification draft.
- `VOCABULARY.md` — canonical terms and allowed meanings.
- `SYMBOLS.md` — symbol table.
- `GRAMMAR.md` — legal statement forms.
- `RELATIONSHIP_VERBS.md` — approved relationship verbs.
- `NAMESPACES.md` — naming and addressing convention.
- `TYPE_SYSTEM.md` — type distinctions and category-error rules.
- `INFERENCE_RULES.md` — first-pass reasoning rules.
- `examples.md` — valid and invalid statement examples.
- `VERSION.md` — current version and stability notes.
- `CHANGELOG.md` — version change history.

## Boundary

ASO-Lang is not the mathematics layer. It is the formal language beneath the mathematics layer.
