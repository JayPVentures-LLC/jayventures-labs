# ASO-Lang Namespaces v0.1

## Purpose

Namespaces give every concept a stable address.

## Namespace pattern

```text
Framework.Layer.Term
```

## Core namespaces

| Namespace | Meaning |
|---|---|
| ASO.Primitive | Primitive ontology concepts. |
| ASO.Function | Derived universal functions. |
| ASO.Theorem | Logical theorem candidates. |
| ASO.Hypothesis | Empirical hypotheses. |
| ASO.Measure | Measurement constructs. |
| IAT.State | Information accessibility states. |
| IAT.Transition | Accessibility transitions. |
| IAT.Phenomenon | Intuition, DejaVu, insight, and related phenomena. |
| Human.Implementation | Human-domain mechanisms. |
| AI.Implementation | Artificial-system mechanisms. |
| Org.Implementation | Organizational mechanisms. |
| Gov.Implementation | Governance mechanisms. |
| RP.Decision | Research-platform decisions. |

## Examples

```text
ASO.Primitive.Existence
ASO.Function.Adaptation
ASO.Theorem.AdaptationRequiresFeedback
IAT.State.LatentInformation
IAT.Phenomenon.Intuition
Human.Implementation.CircadianRhythm
AI.Implementation.WeightUpdate
RP.Decision.PlaceResearchInLabs
```

## Rule

A concept may have one canonical namespace and multiple mapped domain equivalents.

The namespace identifies placement, not truth status.
