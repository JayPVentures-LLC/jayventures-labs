# Research Platform Roadmap

Status: v0.1 operating draft

## Objective

Move foundational frameworks from concept to specification, measurement, simulation, validation, and domain translation.

## Phase 0 — Capture

Goal: preserve emerging ideas without prematurely accepting them.

Deliverables:

- notes
- rough hypotheses
- research questions
- raw observations
- retrospective examples

Exit criteria:

- idea has enough structure to become a proposal or is archived as a note.

## Phase 1 — Proposal

Goal: convert an idea into a reviewable unit.

Deliverables:

- problem statement
- proposed concept or change
- affected framework
- derivability test
- measurement question
- falsification condition
- placement recommendation

Exit criteria:

- proposal is accepted for draft development, rejected, or deferred.

## Phase 2 — Draft Framework

Goal: create a coherent v0.x artifact.

Deliverables:

- canonical paper
- definitions
- diagrams or maps
- boundary statements
- validation plan
- templates if applicable

Exit criteria:

- draft can be reviewed against research standards.

## Phase 3 — Specification

Goal: stabilize terminology and identifiers.

Deliverables:

- spec document
- term registry
- versioning policy
- change-control process
- theorem/hypothesis registry

Exit criteria:

- core definitions are stable within a major version.

## Phase 4 — Measurement

Goal: turn constructs into observable variables.

Deliverables:

- operational definitions
- observable indicators
- scoring rubrics
- reliability protocol
- validity protocol
- limitations

Exit criteria:

- independent researchers can attempt to measure the construct consistently.

## Phase 5 — Simulation

Goal: test formal behavior in controlled environments.

Deliverables:

- agent model
- state model
- feedback engine
- adaptation engine
- logging and visualization

Exit criteria:

- predictions can be tested computationally before applied use.

## Phase 6 — Validation

Goal: test claims against evidence and critique.

Deliverables:

- counterexample tests
- prediction studies
- external review
- failed-case records
- revision proposals

Exit criteria:

- framework is either strengthened, revised, narrowed, or rejected.

## Phase 7 — Atlas and Dictionary

Goal: make the framework usable across domains.

Deliverables:

- domain translation maps
- canonical dictionary entries
- implementation examples
- measurement crosswalks
- open questions

Exit criteria:

- users can compare concepts across domains without collapsing domain-specific mechanisms.

4. Build measurement template.
