# Research Tracks

Status: v0.1 operating draft

## Purpose

Research tracks separate discovery, formalization, measurement, simulation, validation, and application so the platform does not collapse every idea into the same layer.

## Track A — Ontology

Focus:

- primitives
- derived functions
- dependency chains
- concept boundaries
- theorem candidates

Primary outputs:

- specifications
- derivation maps
- term registries

## Track B — Formal Language

Focus:

- canonical vocabulary
- symbols
- grammar
- relationship verbs
- inference rules
- namespaces

Primary outputs:

- ASO-Lang
- IAT-Lang extensions
- machine-readable schemas

## Track routing rule

Every artifact must declare its track. If an artifact spans multiple tracks, it must identify a primary track and secondary tracks.
