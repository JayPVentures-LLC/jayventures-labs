# Information Accessibility Theory
## Validation and Falsification Plan v0.1

## Purpose

IAT must remain open to multiple explanations while requiring evidence for all of them.

The framework strengthens only if it improves classification, measurement, prediction, or explanation of information-accessibility events.

## Competing model classes

### Model A: Existing cognitive mechanisms

Information becomes available through known processes such as memory retrieval, priming, pattern recognition, prediction, incubation, attention shifts, or unconscious integration.

### Model B: Combined known mechanisms

Experiences feel unexplained because several ordinary processes interact in ways that are difficult to introspect.

### Model C: Under-characterized cognitive mechanism

The experience reflects a real cognitive process not yet well described by current models.

### Model D: Unknown information pathway

Information becomes available through a pathway not currently characterized by mainstream scientific models.

### Model E: Coincidence or bias

The experience appears meaningful because of chance, memory reconstruction, selective recall, or confirmation bias.

## Evidence hierarchy

| Evidence Type | Strength |
|---|---|
| Retrospective anecdote | Low; useful for hypothesis generation only. |
| Timestamped personal log before outcome | Moderate. |
| Timestamped log with specific prediction and disconfirmation rule | Stronger. |
| Independent witness or external timestamp | Stronger. |
| Repeated prospective cases with all misses included | Strong. |
| Replication across people or contexts | Very strong. |

## Falsification criteria

IAT weakens if:

- accessibility states cannot be distinguished;
- prospective logs show no pattern beyond chance;
- categories collapse into vague descriptions;
- strong intuitions are no more accurate than weak intuitions;
- confidence is poorly calibrated;
- ordinary explanations account for all cases better than IAT categories;
- the framework cannot predict when information becomes accessible.

## Support criteria

IAT strengthens if:

- accessibility states can be reliably distinguished;
- prospective logs show stable patterns;
- some variables predict accessibility transitions;
- confidence and intensity correlate with outcome in defined categories;
- specific gate conditions predict memory, insight, intuition, or deja vu events;
- unexplained clusters remain after strong ordinary explanations are tested;
- independent reviewers can classify cases consistently.

## Priority measurements

### Accessibility transition type

Which transition occurred?

- latent to accessible
- accessible to conscious
- conscious to expressed
- expressed to applied
- unknown

### Information specificity

How specific was the content?

- vague feeling
- directional urge
- category-level information
- person-specific information
- event-specific information
- time-specific information

### Verification clarity

How clearly can the outcome be scored?

- clear
- partial
- ambiguous
- unverifiable

### Temporal distance

How long between intuition and outcome?

- immediate
- minutes
- hours
- days
- weeks
- months
- unknown

### Alternative explanation strength

Rate ordinary explanations:

0 = none identified  
1 = weak  
2 = plausible  
3 = strong  
4 = highly likely  
5 = sufficient

## Prediction examples

### Prediction 1

If accessibility is affected by sleep state, then intuition and insight events should cluster after sleep, naps, or low-conscious-control states.

### Prediction 2

If emotional salience affects accessibility, then high-stakes interpersonal or safety-related intuitions should show different patterns than neutral guesses.

### Prediction 3

If pattern completion drives some intuitions, then structural intuitions should be more accurate in domains where the subject has repeated exposure and dense prior knowledge.

### Prediction 4

If unknown mechanisms are required, then some prospective cases should remain poorly explained after accounting for cue exposure, prior knowledge, probability, memory reconstruction, and bias.

## Deja vu validation questions

For each deja vu case, record:

1. Was there specific content or only familiarity?
2. Was there an associated prediction?
3. Did the event feel visual, spatial, emotional, verbal, or embodied?
4. Was a prior source later identified?
5. Was the experience followed by useful action or only recognition?
6. Did it occur during fatigue, stress, travel, illness, or sleep disruption?

## Ethical standard

IAT must not encourage unsafe reliance on intuition.

Action guidance:

- Safety-related intuitions may justify cautious action, such as pausing, checking, leaving, calling someone, documenting, or seeking help.
- Intuitions should not be used as sole evidence to accuse, diagnose, punish, or make irreversible decisions about others.
- Unknown does not mean confirmed.

## Current research posture

IAT is a structured observation framework.

It does not prove any mechanism.

It creates a disciplined path for studying experiences where information appears before conscious explanation.
