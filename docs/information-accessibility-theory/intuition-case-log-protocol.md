# Information Accessibility Theory
## Intuition Case Log Protocol v0.1

## Purpose

This protocol turns intuition reports into prospective research records.

The objective is not to prove a preferred explanation. The objective is to document information-accessibility events before outcomes are known whenever possible.

## Core rule

Log the intuition before verification.

Hindsight cases may be recorded as historical examples, but they cannot carry the same evidentiary weight as prospective logs.

## Case categories

### Historical Case

The intuition and outcome are both already known.

Use for pattern discovery only.

### Prospective Case

The intuition is logged before the outcome is known.

Use for accuracy, calibration, and mechanism testing.

### Immediate-Action Case

The intuition requires immediate action before logging is possible.

Log as soon as possible afterward and mark timing clearly.

## Required fields

### Case ID

Unique identifier.

Example:

`IAT-INT-2026-0001`

### Timestamp

Date and time the intuition occurred.

### Context

What was happening?

Include:

- location
- people present
- activity
- recent events
- sleep state
- stress level
- physical state

### Information content

What became available?

Write the exact thought, image, phrase, certainty, or urge if possible.

### Form of intuition

Select all that apply:

- verbal phrase
- image
- physical sensation
- emotional certainty
- urgency
- directional pull
- pattern completion
- knowing without explanation
- dream residue
- deja vu
- other

### Intensity

Rate 1 to 10.

1 = weak impression  
10 = impossible to ignore

### Confidence

Rate expected accuracy from 0% to 100%.

### Action taken

What did you do because of it?

### Predicted outcome

What would count as confirmation?

What would count as disconfirmation?

### Alternative explanations

List plausible ordinary explanations:

- prior knowledge
- subtle cue
- social pattern
- probability
- coincidence
- emotional bias
- memory reconstruction
- other

### Outcome

Record what happened.

Use categories:

- confirmed
- partially confirmed
- not confirmed
- false
- unresolved
- unclear

### Time to verification

How long between intuition and outcome?

### Review notes

What did this case suggest about mechanism, context, or accessibility state?

## Accuracy calculation

Only prospective cases count toward accuracy.

Do not calculate accuracy from dramatic cases only.

Include:

- strong intuitions that were right
- strong intuitions that were wrong
- weak intuitions that were right
- weak intuitions that were wrong
- ignored intuitions
- followed intuitions

## Calibration calculation

Calibration asks whether confidence matches outcomes.

Example:

If 90% confidence cases are confirmed about 90% of the time, calibration is strong.

If 90% confidence cases are confirmed 50% of the time, confidence is inflated.

## Bias controls

### Hindsight bias

Do not rewrite the intuition after the outcome.

### Selection bias

Do not log only dramatic hits.

### Confirmation bias

Record disconfirming outcomes with equal detail.

### Memory drift

Timestamp as close to the event as possible.

## Research use

After sufficient cases, analyze:

- accuracy by category
- accuracy by intensity
- accuracy by domain
- accuracy by physiological state
- accuracy by sleep state
- accuracy by emotional state
- time-to-verification patterns
- false-positive patterns
- cue patterns
- unknown-pattern clusters

## Boundary

This log does not assume intuition is paranormal, supernatural, unconscious cognition, memory processing, coincidence, or any single mechanism.

It records information-accessibility events so competing explanations can be compared.
